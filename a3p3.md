# Assignment 3 Part 3: Context Window Extension via Sequence-Length Curriculum Learning

## Overview

We extend nanochat's context window using a **2-stage sequence-length curriculum learning** approach: first train a picochat model (depth=8, ~42M params) at sequence length 512 (Checkpoint 1), then continue training at sequence length 2048 (Checkpoint 2). The model learns local patterns first at the short context, then extends to leverage long-range dependencies.

---

## Literature Review

### Sequence-Length Curriculum Learning

Curriculum learning (Bengio et al., 2009) structures training from easy to hard examples. When the difficulty measure is **sequence length**, this becomes sequence-length curriculum learning: train on short sequences first, then progressively extend. Several recent works validate this approach:

**[GrowLength: Accelerating LLMs Pretraining by Progressively Growing Training Length](https://arxiv.org/abs/2310.00576)** (Jin et al., 2023)
- Progressively grows training length from 128→256→512→...→4096 over equal-length stages
- Achieves **1.5x faster convergence** and **2-3% lower perplexity** at the same FLOPs budget
- "Training with shorter sequences in early stages is significantly cheaper per step due to quadratic attention complexity, allowing the model to see more tokens in the same wall-clock time"

**[SkyLadder: Better and Faster Pretraining via Context Window Scheduling](https://arxiv.org/abs/2503.15450)** (NeurIPS 2025)
- Expands context window during pretraining from as low as 8 tokens to 32K
- "Models pretrained with shorter context windows consistently outperform their long-context counterparts under a fixed token budget"
- Up to **3.7% gains** on common benchmarks and **22% faster training**
- Recommends allocating **~60% of total training tokens to the short-context phase**
- We follow this recommendation: 3000/5000 steps = 60% at short context, 40% at long context

**[Dataset Decomposition: Faster LLM Training with Variable Sequence Length Curriculum](https://proceedings.neurips.cc/paper_files/paper/2024/file/3f9bf45ea04c98ad7cb857f951f499e2-Paper-Conference.pdf)** (NeurIPS 2024)
- Shows "a correlation between training instability and long sequences, especially at the early stages of training"
- Curriculum starting from short sequence buckets achieves ">2x data efficiency"
- This justifies our Stage 1: training on long sequences when the model is poorly initialized causes instability

**[Variable Sequence Length Training for Long-Context Large Language Models](https://www.cerebras.ai/blog/variable-sequence-length-training-for-long-context-large-language-models)** (Cerebras, March 2025)
- Our **most direct precedent**: two-stage approach with **Stage-1 at 512 tokens, Stage-2 at 2048 tokens**, tested on GPT-3 125M and GPT-3 XL 1.3B
- "Models trained with the VSL training recipe achieve similar evaluation scores as models trained entirely with long sequences"
- We adopt their exact 512→2048 stage design on the same architecture class (GPT)

### Why 512 for Stage 1

- **Cerebras VSL** tested exactly 512→2048 on GPT models and validated it works
- 512 provides enough context for sentence-level and paragraph-level dependencies before extending
- yoyoliuuu's picochat ablation uses `MAX_SEQ_LEN=512`, making our Stage 1 directly comparable to baseline
- 512 is a 4x reduction from 2048, balancing compute savings with sufficient context for learning local patterns

### Context Extension Mechanics

nanochat uses **Rotary Position Embeddings (RoPE)** ([RoFormer](https://arxiv.org/abs/2104.09864), Su et al., 2021), which encode position via rotation matrices with no learned positional parameters. RoPE buffers in nanochat are non-persistent and recomputed on model initialization (`gpt.py` lines 195-199), and all weight tensors have no sequence-length dimension. This means loading a 512-trained checkpoint and setting `--max-seq-len=2048` works directly — RoPE recomputes for the new length, no interpolation or scaling is needed.

We reset the optimizer on extension (`--no-load-optimizer`) because the momentum buffers from the short-context loss landscape are stale for the new regime. This follows standard practice in context extension work (Cerebras VSL).

### Evaluation: Multi-Length BPB

**[What is Wrong with Perplexity for Long-context Language Modeling?](https://arxiv.org/abs/2410.23771)** (Fang et al., 2024)
- Standard perplexity "overlooks key tokens essential for long-context understanding by averaging across all tokens"
- Shows standard PPL has weak correlation with actual long-context performance
- Motivates measuring loss at **multiple context lengths** rather than a single aggregate number

**[Perplexity of fixed-length models](https://huggingface.co/docs/transformers/perplexity)** (HuggingFace)
- Documents the sliding-window strategy for evaluating perplexity at different context lengths
- GPT-2 perplexity varies significantly with evaluation context: PPL 19.44 (no overlap) to 16.44 (stride=512)
- Validates that context length at evaluation time meaningfully affects measured performance

**Why BPB over perplexity:** BPB (bits per byte) is tokenizer-agnostic — it normalizes by bytes rather than tokens, so comparisons remain valid across different tokenizations ([The Gradient: Evaluation Metrics for Language Modeling](https://thegradient.pub/understanding-evaluation-metrics-for-language-models/)). nanochat already uses BPB as its primary loss metric (`nanochat/loss_eval.py`).

**Our evaluation design:** We measure val BPB at sequence lengths [128, 256, 512, 1024, 2048] for both checkpoints. A model that truly leverages long context should show **decreasing BPB as context increases** — more context means better predictions. This multi-length profile reveals:
- Whether the model can leverage progressively longer context
- At what length the model stops benefiting from additional context
- Whether the context extension preserved short-context quality

## Method

**Two-Stage Sequence-Length Curriculum**

1. **Stage 1 → Checkpoint 1:** Train picochat (depth=8, ~42M params) at seq_len=512 for 3000 steps (~60% of total budget). The model learns local linguistic patterns — sentence structure, token co-occurrences, short-range dependencies.

2. **Stage 2 → Checkpoint 2:** Continue training from Checkpoint 1 at seq_len=2048 for 2000 more steps (~40% of budget). Fresh optimizer (no momentum carryover). RoPE recomputes automatically. The model learns long-range dependencies while retaining short-context knowledge.

3. **Evaluation:** Multi-length BPB eval at [128, 256, 512, 1024, 2048] tokens on both checkpoints:
   - **Checkpoint 1** (step 3000): trained at seq_len=512 only — can it use longer context?
   - **Checkpoint 2** (step 5000): extended to seq_len=2048 — did the extension work?

## Training Curves

### `val/bpb` — Validation BPB across both stages

[val/bpb curve from W&B: Stage 1 (picochat-ctx-stage1) + Stage 2 (picochat-ctx-stage2)]

This is the most important curve. Stage 1 (steps 0–3000) shows `val/bpb` decreasing from ~1.5 to ~1.22 as the model learns at seq_len=512. At step 3000, Stage 2 begins at seq_len=2048 with a fresh optimizer. There is a brief spike in `val/bpb` at the transition — this is expected, since the model is now being evaluated on 4x longer sequences that include positions it hasn't seen before. The loss then drops rapidly below Stage 1's final value, reaching ~1.17 by step 5000. The fact that `val/bpb` at 2048 surpasses the Stage 1 minimum (evaluated at 512) confirms the model is successfully leveraging the longer context — it's making better predictions by conditioning on more tokens.

### `train/loss` — Training loss across both stages

[train/loss curve from W&B: Stage 1 (picochat-ctx-stage1) + Stage 2 (picochat-ctx-stage2)]

The training loss follows a similar pattern. Stage 1 shows steady decrease. At the Stage 2 transition (step 3000), training loss jumps because the model now processes 2048-token sequences with long-range dependencies it hasn't learned yet, and the optimizer is reset (no momentum). The loss then decreases throughout Stage 2, indicating the model is learning to handle the longer sequences. The loss does not return to pre-transition levels because the task is harder — predicting tokens conditioned on 2048 positions requires modeling longer-range dependencies than 512.

### `train/lrm` — Learning rate multiplier

[train/lrm curve from W&B: Stage 2 (picochat-ctx-stage2)]

Stage 1 uses `warmdown-ratio=0.0` so the LR is constant (lrm=1.0) throughout — the model trains at full learning rate because training will continue in Stage 2. Stage 2 uses `warmdown-ratio=0.5`, so the LR linearly decays over the final 50% of total iterations (steps 2500–5000). Since Stage 2 resumes at step 3000, the LR is already in the warmdown phase from the start. This is intentional: the model already has a strong foundation from Stage 1 and needs a decaying LR to fine-tune on the new context length without overshooting.

## Multi-Length BPB Evaluation Results

| seq_len | Checkpoint 1 (512-trained, step 3000) | Checkpoint 2 (2048-extended, step 5000) |
|---------|--------------------------------------|----------------------------------------|
| 128     | 1.2447                               | 1.1964                                 |
| 256     | 1.2253                               | 1.1748                                 |
| 512     | 1.2201                               | 1.1655                                 |
| 1024    | 1.2483                               | 1.1606                                 |
| 2048    | 1.3527                               | 1.1675                                 |

### Interpretation

**What the BPB curve means.** A language model with more context should predict the next token better — BPB should decrease as sequence length increases, because the model has more information to condition on. When BPB *increases* with longer context, the model is failing to use that context and the extra positions are introducing noise into the attention computation. This is exactly what Fang et al. (2024) formalize: standard perplexity "overlooks key tokens essential for long-context understanding," and measuring at multiple lengths reveals the model's true effective context window.

**Checkpoint 1 shows a clear context ceiling at 512.** BPB decreases from 128→512 (1.245→1.220), meaning the model learned to condition on up to 512 tokens — its training length. Beyond that, BPB degrades: +0.028 at 1024, +0.133 at 2048. This is the expected failure mode. RoPE encodes position via rotation frequencies that the model has only seen up to position 512 during training. At positions beyond this, the attention scores become out-of-distribution — the model attends to positions it never learned to interpret, which corrupts rather than helps predictions. LongRoPE (Ding et al., 2024) describes this precisely: "extrapolating positional encodings beyond the trained range results in unstable and out-of-distribution attention scores."

**Checkpoint 2 demonstrates successful context extension.** BPB decreases monotonically from 128→1024 (1.196→1.161), a 2.9% improvement from leveraging longer context. This confirms the model learned long-range dependencies during Stage 2 — it can now condition on tokens 500-1000 positions back that Checkpoint 1 couldn't use. The slight uptick at 2048 (1.168 vs 1.161 at 1024) is within normal range; even well-trained models show diminishing returns at the far end of their context window because the marginal value of the 1500th token for predicting the 2049th is small.

**Short-context performance improved, not degraded.** Checkpoint 2 is better than Checkpoint 1 at *every* length, including 128 (1.196 vs 1.245) and 512 (1.166 vs 1.220). This is a 3.9-4.5% improvement across short contexts. This matters because context extension can hurt short-context quality if the model "forgets" local patterns — ProLong (Gao et al., 2024) found that "training only on long data hurts long-context performance" and recommends mixed short+long exposure. Our curriculum approach provides exactly this: Stage 1 builds a strong short-context foundation, and Stage 2 extends without destroying it. The improvement at short contexts comes from the additional 2000 training steps (Stage 2 still trains on short-range patterns embedded within the 2048-token sequences).

**Comparison to Cerebras VSL.** Our results align with the Cerebras Variable Sequence Length findings: "models trained with the VSL training recipe achieve similar evaluation scores as models trained entirely with long sequences." They observed that Stage-1 models (512-only) "only learn to attend to positions up to the short sequence length" — for the first ~700 tokens both models had similar attention patterns, after which the short model deviated. Our BPB degradation at 1024/2048 for Checkpoint 1 is the loss-based manifestation of this same attention pattern breakdown.
